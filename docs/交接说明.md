## 2. 新开发者进入项目

从项目根目录执行：

```bash
pnpm check:env
pnpm install
pnpm services:start
pnpm prisma:generate
pnpm prisma:migrate
pnpm generate:types
```

启动开发服务：

```bash
pnpm dev:backend
pnpm dev:frontend:h5
pnpm dev:admin
```

只检查骨架、不安装依赖：

```bash
SKIP_INSTALL=1 bash scripts/init-project.sh
```

## 3. 开发顺序建议

1. 先执行 `pnpm services:start` 和 `pnpm prisma:migrate` 建立本地数据库。
2. 先实现 Public 接口，确保游客模式可用。
3. 再实现 UserAuth 接口，所有用户数据从 Token 解析 userId。
4. 最后实现 AdminAuth 审核和管理接口。
5. 真实注册、密码校验、刷新 token 和上传文件元信息落库，在当前 Auth 和 Upload 基础上继续补齐。
6. 前端和管理端严格按 `api-contract/openapi.yaml` 对齐字段。

## 4. 权限边界

Public：

- 不需要 Token；
- 不得因为未登录阻断首页、急救知识、医院列表、寻宠详情等公开内容。

UserAuth：

- 必须校验普通用户 Token；
- 后端不得相信前端传来的 userId；
- userId 必须从 Token 解析。

AdminAuth：

- 必须校验管理员 Token；
- 必须校验管理员权限；
- 不允许普通用户访问管理端接口。

## 5. 变更同步要求

新增或修改接口：

- 更新 `docs/接口文档.md`；
- 更新 `api-contract/openapi.yaml`；
- 重新生成 `api-contract/generated-types/api-types.ts`。

修改数据库：

- 修改 `backend/pet-nestjs/prisma/schema.prisma`；
- 增加 Prisma migration；
- 必要时更新 `docs/数据库设计.md`；
- 标明是否需要数据迁移或回滚。

修改登录策略：

- 更新 `README.md`；
- 更新 `docs/项目介绍.md`；
- 更新 `docs/技术架构说明.md`；
- 更新 `AGENTE.md`。

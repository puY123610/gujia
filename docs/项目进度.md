## 下一步

1. 执行 `pnpm services:start` 启动 PostgreSQL/PostGIS、Redis、MinIO。
2. 执行 `pnpm prisma:migrate` 创建本地数据库结构。
3. 从 Public 接口开始实现后端 Controller / Service。
4. 前端按 OpenAPI 生成类型接入 `src/api/`。
5. 管理端从管理员登录和审核列表开始接入。

## 风险和约束

- 用户端不能启动即强制登录；
- Public 接口不能错误加登录守卫；
- UserAuth / AdminAuth 不能直接相信前端 userId；
- 文件本体不能直接存数据库；
- 数据库结构必须以 Prisma schema 和 migration 为唯一来源；
- 接口字段必须以 `api-contract/openapi.yaml` 为唯一契约；
- 敏感配置必须使用环境变量。

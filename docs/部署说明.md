# 部署说明

## 1. 部署目标

本项目第一阶段建议使用 Docker Compose 部署，便于快速上线、测试和迁移。

部署组件包括：

- PostgreSQL；
- Redis / Valkey；
- MinIO；
- NestJS 后端；
- 管理端静态资源；
- H5 静态资源；
- Nginx 反向代理。

## 2. 环境要求

| 环境 | 建议版本 |
|---|---|
| Node.js | 20+ |
| pnpm | 8+ |
| Docker | 24+ |
| Docker Compose | 2+ |
| PostgreSQL | 15+ |
| Redis / Valkey | 7+ |
| Nginx | 1.24+ |

具体版本以项目实际依赖为准。

## 3. 环境变量

后端需要 `.env` 文件，禁止提交真实 `.env` 到仓库。

示例字段：

```txt
NODE_ENV=development
PORT=3000

DATABASE_URL=postgresql://user:password@localhost:5432/pet_platform

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=

JWT_SECRET=<replace-with-secret>
JWT_EXPIRES_IN=7d

STORAGE_DRIVER=minio
MINIO_ENDPOINT=localhost
MINIO_PORT=9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=pet-platform

PUBLIC_FILE_BASE_URL=http://localhost:9000/pet-platform
```

## 4. 本地开发启动

### 4.1 根目录初始化

```bash
pnpm check:env
pnpm install
pnpm services:start
pnpm prisma:generate
pnpm prisma:migrate
pnpm generate:types
```

### 4.2 启动后端

```bash
pnpm dev:backend
```

### 4.3 启动前端用户端

```bash
pnpm dev:frontend:h5
```

微信小程序和 App 运行方式以 uni-app 项目脚本为准。

Android App 首阶段不维护独立原生 Android 工程，走 uni-app App 发行路径；涉及证书、包名、权限申请和原生插件时再单独补充发行文档。

### 4.4 启动管理端

```bash
pnpm dev:admin
```

### 4.5 只启动基础服务

```bash
bash deploy/scripts/start.sh
```

## 5. 登录策略部署影响

用户端支持游客访问，因此前端首页、H5 分享页、公开内容页面不应依赖登录状态才能渲染。

管理端必须启用登录校验，不允许未登录访问管理页面。

后端需要确保：

- Public 接口不要求 Token；
- UserAuth 接口要求普通用户 Token；
- AdminAuth 接口要求管理员 Token 和权限。

## 6. 生产部署建议

生产环境推荐流程：

1. 拉取最新代码；
2. 检查环境变量；
3. 构建后端镜像；
4. 构建管理端静态文件；
5. 构建 H5 静态文件；
6. 启动 Docker Compose；
7. 执行数据库迁移；
8. 检查 Nginx 反向代理；
9. 检查 HTTPS 证书；
10. 验证公开页面、用户登录、管理端登录和文件上传。

当前 Docker Compose 骨架已包含 PostgreSQL/PostGIS、Redis、MinIO、NestJS 后端和 Nginx。执行：

```bash
bash deploy/scripts/deploy.sh
```

会检查环境并按 Compose 构建启动服务。

## 7. Nginx 代理规划

建议路径：

```txt
/api      → NestJS 后端
/admin    → 管理端静态资源
/h5       → H5 静态资源
/files    → 文件访问代理，可选
```

当前 Nginx 配置已落地 `/api`、`/h5`、`/admin`。`/files` 暂不启用独立代理，本地开发阶段文件 URL 由 MinIO 或 `PUBLIC_FILE_BASE_URL` 返回。

## 8. 文件存储部署

开发环境使用 MinIO，便于本地调试上传、下载、预览。

生产环境可选：

- 阿里云 OSS；
- 腾讯云 COS；
- 自建 MinIO。

## 9. 数据库备份

PostgreSQL 必须定期备份。

建议：

- 开发环境：手动备份；
- 测试环境：每日备份；
- 生产环境：每日自动备份，关键版本发布前手动备份。

## 10. 发布检查清单

发布前检查：

- `.env` 配置正确；
- 数据库迁移已执行；
- 后端接口启动正常；
- Public 接口未被登录拦截；
- UserAuth 接口能正确校验 Token；
- AdminAuth 接口能正确校验管理员权限；
- 管理端可访问；
- H5 可访问；
- 文件上传正常；
- Nginx HTTPS 正常；
- 日志无明显错误。

## 11. 回滚原则

如果发布后出现严重问题，应优先回滚：

1. 回滚后端镜像；
2. 回滚前端静态资源；
3. 必要时恢复数据库备份；
4. 记录故障原因；
5. 修复后重新发布。

数据库迁移涉及破坏性操作时，必须提前设计回滚方案。

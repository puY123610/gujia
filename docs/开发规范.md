# 开发规范

## 1. 总原则

1. 代码必须真实可运行。
2. 不写无法落地的伪代码。
3. 不硬编码密钥、密码、Token。
4. 不编造不存在的接口和字段。
5. 修改接口必须同步类型和文档。
6. 修改数据库必须同步迁移和文档。
7. 用户端不允许启动即强制登录。
8. 第一阶段优先清晰、稳定、可维护，不追求过度设计。

## 2. 登录与权限规范

项目统一使用三类权限：

| 权限等级 | 说明 |
|---|---|
| Public | 游客可访问 |
| UserAuth | 普通用户登录后访问 |
| AdminAuth | 管理员登录后访问 |

开发规则：

1. 首页和公开内容不得强制登录。
2. 需要个人数据归属的操作必须登录。
3. 管理端必须登录。
4. 后端必须从 Token 中解析用户身份。
5. 前端登录成功后应返回原目标页面或继续原操作。

## 5. 前端职责规范

### 5.1 页面职责

页面负责：

- 展示数据；
- 接收用户输入；
- 调用接口；
- 处理简单交互。

页面不负责：

- 复杂业务计算；
- 大量接口封装；
- 平台差异封装；
- 全局状态规则。

### 5.2 组件职责

组件应保持可复用。

组件通过 `props` 接收数据，通过 `emit` 向外通知事件。除上传组件、搜索组件等特殊组件外，普通展示组件不直接请求后端接口。

### 5.3 API 调用

所有请求必须经过统一请求封装。

禁止在页面中重复写：

```ts
uni.request(...)
```

应统一放在：

```txt
src/api/
```

### 5.4 登录守卫

需要登录的操作应使用统一登录守卫，例如：

```txt
src/guards/require-login.ts
```

不要在每个页面中重复写不同登录逻辑。

## 8. 后端职责规范

### 8.1 Controller

Controller 只做：

- 接收请求；
- 读取参数；
- 调用 Service；
- 返回结果。

禁止在 Controller 中写复杂业务逻辑。

### 8.2 Service

Service 负责：

- 业务逻辑；
- 权限判断；
- 数据库操作；
- 缓存操作；
- 文件存储操作；
- 事务处理。

### 8.3 DTO

所有请求体必须使用 DTO，并用 `class-validator` 校验。

### 8.4 VO

返回给前端的数据建议使用 VO。不要直接返回包含敏感字段的数据库对象。

## 9. TypeScript 规范

1. 禁止大量使用 `any`。
2. 复杂对象必须定义类型。
3. 接口返回结果必须定义类型。
4. DTO、VO、业务实体必须分清。
5. 函数参数尽量声明类型。
6. 重要函数应声明返回值类型。
7. 不确定数据使用 `unknown`，不要直接用 `any`。

## 10. 数据库规范

1. 数据库结构变更必须走迁移。
2. 核心表必须有创建时间和更新时间。
3. 涉及用户归属的数据必须有用户 ID。
4. 图片、视频、附件不直接存数据库。
5. 状态字段必须有清晰含义。
6. 地理位置数据应保留经纬度，并按需要使用 PostGIS 字段。

## 11. 缓存规范

Redis / Valkey 只保存缓存和临时数据。

允许保存：

- 验证码；
- 热门数据；
- 限流数据；
- 临时锁；
- Token 黑名单。

禁止只把核心业务数据放在缓存中。

## 12. 文件上传规范

1. 后端必须校验文件类型。
2. 后端必须限制文件大小。
3. 禁止上传可执行脚本。
4. 文件上传后返回文件 URL。
5. 数据库保存文件元信息。
6. 删除业务数据时，不一定立即删除文件，但应有后续清理机制。

## 13. Git 提交规范

建议格式：

```txt
feat: 新增功能
fix: 修复问题
docs: 修改文档
style: 代码格式调整
refactor: 重构
test: 测试相关
chore: 构建或配置调整
```

## 14. 工作区与脚本规范

项目根目录使用 pnpm workspace 统一管理三端和接口契约。

常用命令：

```bash
pnpm check:env
pnpm install
pnpm services:start
pnpm generate:types
pnpm dev:backend
pnpm dev:frontend:h5
pnpm dev:admin
```

脚本约定：

| 脚本 | 用途 |
|---|---|
| scripts/check-env.sh | 检查 Node、pnpm、Docker、关键目录和配置 |
| scripts/init-project.sh | 初始化本地环境文件并安装依赖 |
| scripts/generate-types.sh | 根据 OpenAPI 生成 TypeScript 类型 |
| deploy/scripts/start.sh | 启动本地基础服务 |
| deploy/scripts/stop.sh | 停止本地服务 |
| deploy/scripts/deploy.sh | 按 Docker Compose 构建并启动部署骨架 |
| deploy/scripts/backup.sh | 导出 PostgreSQL 备份 |

## 15. 安全规范

1. 密码必须哈希保存。
2. JWT 密钥放在环境变量。
3. 数据库密码放在环境变量。
4. 对象存储密钥放在环境变量。
5. 用户 ID 从 Token 中解析，不直接相信前端传参。
6. 管理端接口必须做权限校验。
7. 所有输入都必须校验。
8. 上传文件必须校验大小和类型。

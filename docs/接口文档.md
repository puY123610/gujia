# 接口文档

## 1. 接口说明

本文档用于约定前端、管理端与后端之间的 HTTP API。所有接口由 NestJS 后端提供。

机器可读契约位于：

```txt
api-contract/openapi.yaml
```

前端和管理端类型基线位于：

```txt
api-contract/generated-types/api-types.ts
```

安装依赖后从根目录执行：

```bash
pnpm generate:types
```

基础路径建议：

```txt
开发环境：http://localhost:3000/api
生产环境：https://api.example.com/api
```

实际域名以部署配置为准。

## 2. 权限等级

| 权限等级 | 说明 |
|---|---|
| Public | 游客可访问，不需要 Token |
| UserAuth | 普通用户登录后访问，需要用户 Token |
| AdminAuth | 管理员访问，需要管理员 Token 和权限 |

## 3. 通用请求头

UserAuth 和 AdminAuth 接口需要：

```txt
Authorization: Bearer <token>
Content-Type: application/json
```

文件上传接口使用：

```txt
Content-Type: multipart/form-data
```

## 4. 统一返回格式

成功：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

失败：

```json
{
  "code": 400,
  "message": "请求参数错误",
  "data": null
}
```

## 5. 通用错误码

| code | 含义 |
|---|---|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 无效 |
| 403 | 无权限 |
| 404 | 数据不存在 |
| 409 | 数据冲突 |
| 500 | 服务端错误 |

## 7. 用户接口 UserAuth

### 7.1 用户登录

```txt
POST /auth/login
```

权限等级：

```txt
Public
```

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| phone | string | 是 | 手机号 |
| password | string | 是 | 密码 |

返回：

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "jwt-token",
    "user": {
      "id": "1",
      "nickname": "用户昵称",
      "avatarUrl": "https://example.com/avatar.png"
    }
  }
}
```

## 9. 接口变更规则

1. 新增接口必须更新本文档。
2. 每个接口必须标记权限等级。
3. 修改字段必须同步更新前端类型定义。
4. 删除接口必须确认前端和管理端不再使用。
5. 接口字段统一使用小驼峰命名。
